## Rules learned
1. **API date vs user deadline mismatch**: The Aviasales cached API returns cheapest-of-month dates, not cheapest-in-window. When the user's deadline is early in the month, ALL routes can be rejected. Fix: fall back to computed departure date when API date is out of window.
2. **Island origins need direct-to-EU segments**: Manila is an island â€” no ground connections to hubs. Without a direct MNLâ†’LON segment, the only options are 2+ stops. Adding the nonstop dramatically improves results.
3. **Philippines connectivity**: MNL has strong connections to ICN/NRT that were missing. These open up Korean Air and ANA/JAL routing options.

## Contradictions with existing rules
- The date filter at `buildRouteFromEdges:1225` was introduced by bug fix #13 to "reject routes with incoherent flight leg dates." The new fallback logic preserves this intent (still rejects if fallback date is also bad) but relaxes it for cases where the API returns out-of-window dates. The `dateIncoherent` check at lines 1231-1255 still catches multi-leg date mismatches separately â€” no regression there.
